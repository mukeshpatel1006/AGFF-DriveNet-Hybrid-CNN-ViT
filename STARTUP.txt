⚙️ SYSTEM SPECIFICATIONS
═══════════════════════════════════════════════════════════════════════════

Model Architecture:
  • CNN Backbone: EfficientNet-B0 (1.28M params)
  • ViT Backbone: TinyViT (lightweight)
  • Fusion: AGFF (Adaptive Gated Feature Fusion)
  • Temporal: BiLSTM (2 layers, bidirectional)
  • Input: 128×128×3 RGB images
  • Sequence: 4 temporal frames
  • Output: 4-class probabilities (softmax)

Performance (GPU - RTX 3090):
  • Load Time: 5-15 seconds
  • Inference: 60+ FPS
  • Latency: 15-20ms per frame
  • Memory: ~800MB VRAM

Performance (CPU - Intel i7+):
  • Load Time: 20-40 seconds
  • Inference: 10-15 FPS
  • Latency: 70-100ms per frame
  • Memory: ~1.5GB RAM

🔧 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

"Webcam not accessible"
  → Check Windows Settings > Privacy & Security > Camera
  → Close other apps using the webcam
  → Try USB camera instead of built-in

"Very slow performance"
  → Running on CPU? Normal behavior (10-15 FPS)
  → Close other applications
  → Reduce screen resolution
  → Use GPU-enabled device

"Model not loading"
  → Verify agff_hybrid_model.pth exists (43.14 MB)
  → System will use randomly initialized model as fallback
  → Application continues to work with randomized predictions

"Port already in use (Address already in use)"
  → Kill existing Streamlit process
  → Or use: streamlit run app.py --server.port 8502

For more help, see QUICK_START.md Troubleshooting section
